Modelo OSI

Modelo de referencias OSI (Interconexión de Sistemas Abiertos)

Principios.

Crear capa en donde se requiera nivel diferente de abstracción

Cada capa debe tener una función definida

Elegir los limites de las capas que minimice el flujo de información a través de las interfaces

La cantidad de capas debe ser suficientes para no agrupar funciones distintas en la misma capa y lo bastante pequeña para que sea manejable la arquitectura

  1. Capa física

Se Transmisión de bits puros a través de un canal de transmisión, el diseño esta relacionado con asegurarse que cuando un lado envíe un bit 1 el otro lado reciba un bit 1.

  1. Capa de enlace de datos

Transformar un medio de transmisión puro en una línea libre de errores de trasmisión.

El emisor divide los datos de entrada en tramas de datos que transmite en forma secuencial

Para confirmar la recepción de correcta de cada trama el receptor devuelve una trama de confirmación de recepción.

Control de acceso al medio: resuelve el problema de conyrolar el acceso al canal compartido.

  1. Capa de red

Controla la operación de la subred. Determina cómo se encaminan los paquetes desde el origen hasta el destino.

Las rutas se pueden basar en tablas estáticas que se “codifican” en la red y rara vez cambian, aunque es más común que se actualicen de manera automática para evitar las fallas en los componentes.

  1. Capa de transporte

Su función es aceptar datos de la capa superior, dividirlos en unidades más pequeñas si es necesario, pasar estos datos a la capa de red y asegurar que todas las piezas lleguen correctamente al otro extremo.

Controla las comunicaciones de extremo a extremo entre dos dispositivos

  1. Capa de sesión

Permite al usuario establecer conexión entre diferentes máquinas, por ejemplo.

El control de diálogos, el manejo de tokens para evitar que dos partes realicen la misma acción critica simultáneamente, la sincronización usa puntos de referencia en las transmisiones extensas para reanudar desde el último punto de referencia en caso de una interrupción.

  1. Capa de presentación

Se enfoca en la sintaxis y la semántica de la información transmitida. Para hacer posible la comunicación entre computadoras con distintas representaciones internas de datos, en otras palabras prepara los datos para su consumo por las aplicaciones.

Es la responsable de traducir el cifrado y la compresión de los datos. 

  1. Capa de aplicación

Es la única capa que interactúa directamente con los datos del usuario. permitir a los usuarios ejecutar acciones en sus propias aplicaciones y permite la comunicación entre el resto de capas inferiores.

Modelo TCP

Siglas de Transmission Control Protocol/Internet Protocol (Protocolo de control de transmisión/Protocolo de Internet).

Identificación del grupo de protocolos de red para la transferencia de datos en redes, entre equipos informáticos e internet.

TCP es el Protocolo de Control de Transmisión que permite establecer una conexión y el intercambio de datos entre dos anfitriones.

IP o protocolo de internet, utiliza direcciones series de cuatro octetos con formato de punto decimal.

  1. Capa de enlace

Es la primera capa del modelo y ofrece la posibilidad de acceso físico a la red

  1. Capa de red o Internet

Proporciona el paquete de datos y administra las direcciones IP.

  1. Capa de Transporte

Nos permite conocer el estado de la transmisión, así como los datos de enrutamiento para asociar un tipo de aplicación con un tipo de dato.

  1. Capa de Aplicación

Es el grupo de aplicaciones que requiere comunicación de red, es con lo que se relaciona el usuario.
